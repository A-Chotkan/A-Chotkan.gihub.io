STARR

Ik ga mijn eindopdracht hebben over mijn project boter kaas en eieren en dit ga ik vertellen via de methode STARRT.

Situatie (S):
In periode 2 hadden we een opdracht gekregen om het spel boter kaas en eieren te maken, en dit deed ik alleen dus moest ik alleen het spel boter kaas en eieren maken en dit moest voor het vak project en dus werkte ik tijdens de lessen van project aan dit opdracht en ik werkte thuis nog verder aan mijn project als ik iets nog niet af had.
Taak (T):
Mijn taak was om het spel helemaal af te krijgen op de oplever datum en het spel moest er heel mooi uit zien en het belangrijkste het moest werken. Over de taken die ik had, ik zag zelf dat ik alles moest doen, want ik werkte alleen dus moest ik alles zelf gaan doen bijvoorbeeld: een sitemap, wireframe, mockup, plan van aanpak en logboek maken, maar ik zag dat als geen probleem, want ik vond het belangrijkste dat je het goed moet in plannen wanneer je dat gaat doen, want als je alleen werkt dan moet je natuurlijk meer doen dan als je in een groepje werkt. De rol die ik mezelf geef is denk ik een alles doener, want ik moest alles alleen doen en niemand die iets voor mij ging maken, maar daar koos ik ook zelf voor en daar heb ik geen spijt van.
Aanpak (A):
Ik vond een aanpak maken heel belangrijk, want dan heb je een idee hoe je het moet maken. Als eerste ging ik denken van hoe het eruit moet zien dus moesten we een wireframe en mockup maken, en dat deed ik, dus had ik ook een beeld hoe mijn spel eruit moest zien, en we kregen te horen wat er allemaal in mijn spel moest, dus dat paste ik nog aan in mijn wireframe en mockup. Toen ik mijn mockup/kleurschema ging maken, had ik bepaalde kleuren gekozen maar uiteindelijk toen ik klaar was had ik toch andere kleuren gekozen. En ik had het uiteindelijk toen ik klaar was ook anders ingedeeld, want naar mijn mening is het precies volgen van de mockup en wireframe best lastig want als je het aan het maken bent dan zie je ook dat er andere mooiere opties zijn. Maar ik wist eerlijk gezegd eerst niet hoe ik moest beginnen, want ik was toen der tijd niet zo goed met Java Script dus bij het begin kreeg ik een beetje error. Dus begon ik als eerst met het maken van de 12 boxen die je moest hebben voor je spel, en het inrichten van wat waar moet staan. Toen ik best wel een begin had gemaakt met het stijlen en alles op de juiste plek te zetten, was het maar eens tijd om een begin te maken met Java Script, dus ik begon met het maken van alerts die de speler vroeg hoe de spelers heten en dat die twee dus tegen elkaar spelen. In het begin vond ik het ook best moeilijk maar ik had ook veel uitleg video's gekeken van Java Script en had ook veel gevraagd aan me klas genoten en Java Script en project docent en toen was het uiteindelijk groten deels wel gelukt. Ik had alleen me spel niet helemaal compleet want ik had geen scoreboard gemaakt, want daar had ik toen geen tijd meer voor om te maken, want het was bijna inlever tijd en dus had ik die niet meer erbij gemaakt want dat zou ik niet meer redden met tijd. Maar ik had mijn spel voor de rest wel af gelukkig.
Resultaat (R):
Uiteindelijk was ik klaar en moest ik mijn spel gaan inleveren dus dat deed ik ook en wel met een redelijk goed gevoel, want mijn spel werkte wel en zag er ook naar mijn mening goed uit. Ik had als kleuren schema dat het niet te drukke kleuren moest zijn dus had ik gekozen voor 2 simpele kleuren en ik had ook een restart knop en had ook dat het van zelf aangeeft wie heeft gewonnen of verloren of gelijk spel. Om je cijfer te krijgen moest je met een docent zitten en je spel en je code laten zien. Er werd gezegd dat me spel er goed uitziet alleen een min punt was dat ik de scoreboard niet erbij had en dat ik weinig commentaar had en dat in de code paar fouten zaten maar hij vond het spel wel goed eruit zien en het deed het ook. En als cijfer kreeg ik ook een voldoende en daar was ik ook tevreden mee, want ik had wel me best gedaan en er moeite in gestopt dus ik was wel naar mijn mening tevreden.
Reflectie (R):
Ik was op het moment zelf zeker tevreden op mijn spel, want we zaten in de 2de periode dus had nog niet zo veel kennis als nu, maar op dat moment was ik zeker tevreden op het eind resultaat, want er zat ook veel moeite in en ik had er ook heel veel van geleerd, met HTML en Java Script, want zonder uitleg zou ik dit niet kunnen maken. Als ik er nu terug kijk op mijn resultaat dan zie ik dat ik wel anders zou kunnen doen, bijvoorbeeld: met hoe het eruit ziet, me code netter hebben en meer commentaar toevoegen. Ik zou het ook anders hebben gemaakt nu, want nu weet ik al veel meer over HTML en Java Script dus zou ik het eigenlijk nu anders aanpakken, maar vorige periode wist ik nog niet hoe ik het anders zou moesten maken, dus ik ben wel tevreden met mijn eind resultaat.
